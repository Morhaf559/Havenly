# تقرير تحليل مشروع My Havenly Application

## 📋 نظرة عامة

**My Havenly Application** هو تطبيق Flutter لإدارة الشقق العقارية، يتيح للمستخدمين استئجار الشقق وإدارتها. التطبيق يدعم نظام مزدوج للمستخدمين (مالكين ومستأجرين) مع واجهات متخصصة لكل نوع.

### بنية كل Feature

كل feature يتبع نفس البنية الموحدة:

```
feature_name/
├── bindings/              # Dependency Injection للـ feature
├── controllers/           # Controllers لإدارة الحالة
├── endpoints/             # تعريفات API endpoints
├── models/                # نماذج البيانات الخاصة بالـ feature

├── views/                 # واجهات المستخدم (Screens)
└── widgets/               # Widgets خاصة بالـ feature (إن وجدت)
```

## 🏛️ المعمارية المستخدمة

### 1. Clean Architecture مع Feature-Based Structure

المشروع يستخدم **Clean Architecture** مع تنظيم **Feature-Based**:

```
┌─────────────────────────────────────────┐
│           Presentation Layer             │
│  (Views/Screens, Widgets, Controllers)  │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│          Business Logic Layer           │
│         (Repositories, Services)        │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│            Data Layer                   │
│    (API Services, Endpoints, Models)   │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         Infrastructure Layer             │
│  (Dio Client, Storage, Network Utils)    │
└─────────────────────────────────────────┘
```

### 2. طبقات المعمارية

#### أ) Presentation Layer
- **Views/Screens**: واجهات المستخدم
- **Controllers**: إدارة الحالة باستخدام GetX
- **Widgets**: مكونات UI قابلة لإعادة الاستخدام

#### ب) Business Logic Layer
- **Repositories**: 
  - معالجة البيانات وتحويلها
  - تطبيق الفلاتر والبحث
  - تحويل البيانات بين Formats (String ↔ Number)
  - معالجة الحقول متعددة اللغات
  
- **Services**: 
  - منطق العمل الإضافي
  - معالجة البيانات قبل الإرسال

#### ج) Data Layer
- **API Services**: 
  - استدعاءات HTTP مباشرة
  - استخدام ApiService الموحد
  - بناء Query Parameters
  
- **Endpoints**: 
  - تعريفات مسارات API
  - توثيق Endpoints
  
- **Models**: 
  - نماذج البيانات
  - Serialization/Deserialization

#### د) Infrastructure Layer
- **Dio Client**: HTTP client مع Interceptors
- **Storage Service**: التخزين المحلي
- **Network Utils**: أدوات مساعدة للشبكة

### 3. Dependency Injection

يستخدم المشروع **GetX Dependency Injection**:

- **CoreBinding**: تسجيل Controllers الأساسية (permanent)
- **Feature Bindings**: تسجيل Controllers الخاصة بكل feature (lazy)

### 4. State Management Pattern

- **GetX Reactive Programming**: 
  - استخدام `.obs` للـ Observables
  - `Obx()` و `GetBuilder()` للـ UI updates
  - `ever()` للاستماع للتغييرات

### 5. Error Handling

- **ApiException Hierarchy**: 
  - `AuthenticationException`
  - `ServerException`
  - `NetworkException`
  - `ValidationException`
  - `UnknownException`

- **Error Interceptors**: معالجة الأخطاء تلقائياً في Interceptor

### 6. Localization Strategy

- دعم متعدد اللغات (عربي/إنجليزي)
- الحقول متعددة اللغات: `{ "en": "...", "ar": "..." }`
- `LocaleController` لإدارة اللغة الحالية
- `LanguageInterceptor` لإضافة `Accept-Language` header

### 7. Authentication Flow

```
1. User Login/Register
   ↓
2. AuthStateController.setAuthState(token, user)
   ↓
3. Token stored in Storage + Memory
   ↓
4. AuthInterceptor adds token to all requests
   ↓
5. On 401: Auto logout + redirect to login
```

#### 3. إنشاء Models

```dart
// models/new_feature_model.dart
class NewFeatureModel {
  // Properties
  // fromJson, toJson methods

```

#### 4. إنشاء API Service

```dart
// services/new_feature_api_service.dart
class NewFeatureApiService {
  Future<ApiResponse<NewFeatureModel>> getList() async {
    return await ApiService.getPaginated<NewFeatureModel>(
      path: NewFeatureEndpoints.list,
      fromJson: (json) => NewFeatureModel.fromJson(json),
    );
  }
}
```

#### 5. إنشاء Repository

```dart
// repositories/new_feature_repository.dart
class NewFeatureRepository {
  final NewFeatureApiService _apiService = Get.find();
  
  Future<ApiResponse<NewFeatureModel>> getList() async {
    // Business logic
    // Data transformation
    // Filtering, sorting
    return await _apiService.getList();
  }
}
```

#### 6. إنشاء Controller

```dart
// controllers/new_feature_controller.dart
class NewFeatureController extends BaseController {
  final NewFeatureRepository _repository = Get.find();
  
  final RxList<NewFeatureModel> items = <NewFeatureModel>[].obs;
  
  @override
  void onInit() {
    super.onInit();
    loadData();
  }
  
  Future<void> loadData() async {
    try {
      setLoading(true);
      final response = await _repository.getList();
      items.value = response.data;
    } catch (e) {
      handleError(e);
    } finally {
      setLoading(false);
    }
  }
}
```

#### 7. إنشاء Binding

```dart
// bindings/new_feature_binding.dart
class NewFeatureBinding extends Bindings {
  @override
  void dependencies() {
    Get.lazyPut<NewFeatureApiService>(() => NewFeatureApiService());
    Get.lazyPut<NewFeatureRepository>(() => NewFeatureRepository());
    Get.lazyPut<NewFeatureController>(() => NewFeatureController());
  }
}
```

#### 8. إنشاء Screen

```dart
// views/screens/new_feature_screen.dart
class NewFeatureScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final controller = Get.find<NewFeatureController>();
    
    return Scaffold(
      body: Obx(() {
        if (controller.isLoading.value) {
          return LoadingWidget();
        }
        if (controller.errorMessage.value != null) {
          return ErrorStateWidget(message: controller.errorMessage.value!);
        }
        return ListView.builder(
          itemCount: controller.items.length,
          itemBuilder: (context, index) {
            return ItemWidget(item: controller.items[index]);
          },
        );
      }),
    );
  }
}
```

#### 9. إضافة Route
